What the receipt captures

Greedy segmenter

Topological walk with YAML breaker rules — deny ops, FP32 islands, backend switches.

Parity probes

Per-output tolerance report vs golden ORT execution (CPU baseline).

Latency budget

Per-segment estimates with backend multipliers (TRT, ORT, custom-kernel).

Early publish

Priority ordering for downstream consumers — don't wait for the full graph.

Multi-GPU splits

Cross-gpu-copy insertion points for pipeline parallelism.

Deterministic JSON

CI-friendly artifacts: plan → run → report, same discipline as eval workbenches.

Quick start

pip install nuro-ftl-segment-receipts[onnx]

segment-receipts plan model.onnx -r rules.yaml
segment-receipts run model.onnx -r rules.yaml -o receipts/run1
segment-receipts report receipts/run1/receipt.json
