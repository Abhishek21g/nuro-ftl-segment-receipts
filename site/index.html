Company problem · Nuro FTL blog

Silent compiler regressions don't crash. They drift.

Nuro built FTL segment breakers because TensorRT and FP16 paths change numerics without errors. This tool augments your ONNX graph, diffs every intermediate activation, pinpoints the first failing node, and emits breaker YAML — the pre-deploy audit their public blog describes but doesn't ship.

"Third party compilers are notorious for causing silent regressions in the model's performance." — Nuro FTL blog

60-second demo for a hiring manager

No GPU. No Nuro credentials. Real ORT intermediate tensor scan on bundled branch.onnx.

pip install nuro-ftl-segment-receipts[onnx]

segment-receipts run examples/models/branch.onnx -o out/receipts
segment-receipts doctor out/receipts/<run-id>
open out/receipts/<run-id>/regression_report.html
